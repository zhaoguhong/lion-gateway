# 插件开发

Lion Gateway 使用插件化架构，可以通过开发自定义插件来扩展功能。

## 插件架构

```
PluginHandler (接口)
       ↑
AbstractPluginHandler (抽象基类)
       ↑
   你的插件
```

## 开发步骤

### 1. 定义插件枚举

在 `PluginEnum.java` 中添加：

```java
public enum PluginEnum {
    WAF("waf", "WAF 插件"),
    HTTP("http", "HTTP 插件"),
    MY_PLUGIN("myPlugin", "我的插件"); // 新增

    // ...
}
```

### 2. 创建插件类

继承 `AbstractPluginHandler`：

```java
@Component
public class MyPluginHandler extends AbstractPluginHandler {

    @Override
    public PluginEnum getPlugin() {
        return PluginEnum.MY_PLUGIN;
    }

    @Override
    protected void doHandle(RequestContext context) {
        // 插件逻辑
        String header = context.getHeaders().get("X-My-Header");
        context.setResponseContent("Processed by MyPlugin");
    }
}
```

### 3. 配置插件

在 `application.yml` 中配置：

```yaml
lion:
  plugins:
    myPlugin:
      enabled: true
      rules:
        - name: myRule
          matchMode: and
          conditions:
            - header,X-My-Header|=|test
```

## 核心接口

### PluginHandler
```java
public interface PluginHandler {
    void handle(RequestContext context);
    PluginEnum getPlugin();
    int order();
}
```

### AbstractPluginHandler
提供了模板方法：
- `handle()` - 入口，检查启用状态、规则匹配
- `doHandle()` - 子类实现具体逻辑
- `getPlugin()` - 返回插件枚举
- `order()` - 执行顺序，默认 0

## 规则匹配

### 匹配模式

| 模式 | 说明 |
|------|------|
| and | 所有条件都满足 |
| or | 任一条件满足 |
| none | 所有条件都不满足 |

### 条件格式

```
数据源,键|操作|值
```

示例：
```
header,X-User|=|admin
uri,/api/|!=|/api/internal
cookie,sessionId|=|abc123
```

### 数据源扩展

实现 `RequestDataLoader` 接口：

```java
@Component
public class CookieRequestDataLoader implements RequestDataLoader {

    @Override
    public RequestDataEnum getDataMode() {
        return RequestDataEnum.COOKIE;
    }

    @Override
    public String load(RequestContext context, String key) {
        // 从 Cookie 中读取值
        return context.getCookies().get(key);
    }
}
```

并在 `RequestDataEnum` 中添加枚举。

### 断言扩展

实现 `Predicate` 接口：

```java
@Component
public class ContainsPredicate implements Predicate {

    @Override
    public PredicateEnum getPredicate() {
        return PredicateEnum.CONTAINS;
    }

    @Override
    public boolean match(String actual, String expected) {
        return actual != null && actual.contains(expected);
    }
}
```

并在 `PredicateEnum` 中添加枚举。

## 插件顺序

通过 `order()` 方法指定执行顺序，数值越小优先级越高：

```java
@Override
public int order() {
    return 10; // 比默认插件晚执行
}
```

## 示例插件

### WafPluginHandler
- 用途：Web 应用防火墙
- 功能：拦截恶意请求

### HttpPluginHandler
- 用途：HTTP 请求转发
- 特性：使用虚拟线程 + HttpClient

## 测试插件

编写单元测试：

```java
@Test
void testMyPlugin() {
    RequestContext context = RequestContext.builder()
        .uri("/test")
        .header("X-My-Header", "test")
        .build();

    myPluginHandler.handle(context);

    assertEquals("Processed by MyPlugin", context.getResponseContent());
}
```
