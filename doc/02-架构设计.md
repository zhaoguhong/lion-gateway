# 架构设计

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        请求流量                               │
└────────────────────────────┬────────────────────────────────┘
                             │
                    ┌────────▼────────┐
                    │   Netty Server  │  (9999端口)
                    │  - Boss Group   │
                    │  - Worker Group │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ HttpRequest     │
                    │ Handler         │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  Handler Chain  │  责任链
                    │  ┌───────────┐ │
                    │  │ WAF Plugin│ │
                    │  └─────┬─────┘ │
                    │  ┌─────▼─────┐ │
                    │  │HTTP Plugin│ │
                    │  └─────┬─────┘ │
                    │  ┌─────▼─────┐ │
                    │  │  ...      │ │
                    │  └───────────┘ │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │    响应返回      │
                    └─────────────────┘
```

## 核心模块

### 1. Netty 网络层 (`netty/`)

#### NettyServer
- Boss 线程组：1 个线程，处理连接请求
- Worker 线程组：CPU 核心数 + 1，处理 I/O
- 监听 9999 端口

#### LionGatewayChannelInitializer
- 初始化 ChannelPipeline
- 添加异常处理器、HTTP 编解码器、请求处理器

#### HttpRequestHandler
- 解析 HTTP 请求（GET/POST）
- 提取请求参数（URL、JSON、表单）
- 构建 RequestContext
- 调用责任链处理

### 2. 核心上下文 (`core/`)

#### RequestContext
请求上下文，包含：
- HTTP 方法
- URI
- Header
- 请求参数
- 响应内容
- 异常信息

使用 Lombok @Builder 构建。

### 3. 配置层 (`config/`)

#### LionProperties
配置前缀 `lion`，包含插件列表。

#### PluginConfig
插件配置：
- enabled：是否启用
- rules：规则列表

#### RuleConfig
规则配置：
- name：规则名称
- matchMode：匹配模式（and/or/none）
- conditions：条件列表

#### VirtualThreadConfig
虚拟线程配置：
- 配置 Spring 异步任务执行器
- 提供 virtualThreadExecutor Bean

### 4. 插件系统 (`plugin/`)

#### 责任链模式
```
HandlerChain (接口)
    ↓
DefaultHandlerChain (实现)
    ↓
PluginHandler 1 → PluginHandler 2 → ...
```

#### 插件处理器
```
PluginHandler (接口)
    ↓
AbstractPluginHandler (抽象基类)
    ↓
┌─────────────┬─────────────┐
WafPlugin   HttpPlugin   ...
```

#### 规则匹配流程

```
插件启用?
    ↓ (否)
   跳过
    ↓ (是)
遍历规则
    ↓
每个规则: 检查条件
    ↓
匹配模式: AND/OR/NONE
    ↓
匹配成功?
    ↓ (是)
执行插件逻辑
```

#### 条件格式
```
header,aaa|=|bbb
  ↓    ↓  ↓  ↓
数据源 键  操作 值
```

支持的数据源：header、uri、cookie、method 等

支持的操作：
| 操作 | 说明 |
|------|------|
| `=` | 等于 |
| `!=` | 不等于 |
| `>` | 大于 |
| `<` | 小于 |
| `>=` | 大于等于 |
| `<=` | 小于等于 |
| `contains` | 包含 |
| `notContains` | 不包含 |
| `startsWith` | 开头匹配 |
| `endsWith` | 结尾匹配 |
| `regex` | 正则匹配 |
| `in` | 在列表中 |
| `empty` | 为空 |
| `notEmpty` | 不为空 |

支持的匹配模式：and（全部匹配）、or（任一匹配）、none（都不匹配）

## 设计模式

| 模式 | 应用 | 说明 |
|------|------|------|
| 责任链 | HandlerChain | 插件链式处理 |
| 工厂 | *Factory | Predicate/MatchMode/Loader |
| 策略 | MatchMode | AllMatch/AnyMatch/NoneMatch |
| 模板方法 | AbstractPluginHandler | 定义处理流程 |
| 构建器 | RequestContext | 上下文构建 |

## 虚拟线程最佳实践

### 推荐做法
- ✅ 直接编写同步阻塞代码
- ✅ 使用 ReentrantLock 替代 synchronized
- ✅ 大量并发 I/O 操作

### 避免做法
- ❌ 使用 synchronized（会固定到平台线程）
- ❌ 池化虚拟线程（没必要）
- ❌ CPU 密集型任务长时间占用

### 示例
```java
// 推荐
@Autowired
private Executor virtualThreadExecutor;

public void doSomething() {
    virtualThreadExecutor.execute(() -> {
        // 阻塞式调用，自动切换到虚拟线程
        String response = restTemplate.getForObject(url, String.class);
    });
}
```
