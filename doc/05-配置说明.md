# 配置说明

## application.yml

主配置文件位置：`lion-gateway-server/src/main/resources/application.yml`

## 完整配置示例

```yaml
spring:
  application:
    name: lion-gateway-server

  # 虚拟线程配置
  threads:
    virtual:
      enabled: true

# Lion Gateway 配置
lion:
  plugins:
    # WAF 插件
    waf:
      enabled: true
      rules:
        - name: blockMalicious
          matchMode: or
          conditions:
            - header,User-Agent|=|bad-bot
            - uri,/admin|=|/admin

    # HTTP 插件
    http:
      enabled: true
      rules:
        - name: forwardApi
          matchMode: and
          conditions:
            - header,X-Forward|=|true
```

## 配置项详解

### spring.threads.virtual.enabled

| 类型 | 默认值 | 说明 |
|------|--------|------|
| boolean | true | 是否启用虚拟线程 |

### lion.plugins

插件配置，格式：
```yaml
lion:
  plugins:
    <插件名>:
      enabled: <boolean>
      rules:
        - name: <规则名>
          matchMode: <and|or|none>
          conditions:
            - <条件1>
            - <条件2>
```

### 插件配置项

| 配置项 | 类型 | 说明 |
|--------|------|------|
| enabled | boolean | 是否启用该插件 |
| rules | array | 规则列表 |

### 规则配置项

| 配置项 | 类型 | 说明 |
|--------|------|------|
| name | string | 规则名称 |
| matchMode | string | 匹配模式：and/or/none |
| conditions | array | 条件列表 |

### 条件格式

```
数据源,键|操作|值
```

#### 数据源

| 数据源 | 说明 |
|--------|------|
| header | HTTP 请求头 |
| uri | 请求 URI |
| cookie | Cookie（需扩展） |

#### 操作

| 操作 | 说明 |
|------|------|
| = | 等于 |
| != | 不等于 |

#### 示例

```yaml
# Header aaa 等于 bbb
- header,aaa|=|bbb

# URI 不等于 /internal
- uri,/api|!=|/internal
```

## 代码配置类

### LionProperties

配置前缀 `lion`，注入所有插件配置。

### PluginConfigManager

管理插件配置，提供：
- `getPluginConfig(PluginEnum)` - 获取插件配置
- `isEnabled(PluginEnum)` - 检查插件是否启用

## Netty 配置

Netty 端口在 `NettyServer.java` 中硬编码：

```java
int port = 9999;
```

线程组配置：
- Boss：1 个线程
- Worker：CPU 核心数 + 1

## 虚拟线程配置

`VirtualThreadConfig.java` 提供：
- Spring 异步任务执行器（使用虚拟线程）
- `virtualThreadExecutor` Bean（可直接注入使用）

## 环境变量

可通过环境变量覆盖配置：

```bash
export LION_PLUGINS_WAF_ENABLED=false
java -jar lion-gateway-server.jar
```
