# 项目介绍

## 项目概述

**Lion Gateway** 是一个轻量级、高性能的 API 网关，取名 "Lion（狮子）"，寓意像中国住宅门前的守护神石狮子一样守护后端服务。

该项目最初使用 JDK 8 + Reactor 响应式编程模型，后升级到 JDK 21 + 虚拟线程，大幅简化了并发编程模型。

## 核心特性

### 1. 高性能网络框架
- 基于 Netty 4.1 实现高性能网络通信
- Boss/Worker 线程组模型
- 监听 9999 端口

### 2. 虚拟线程并发
- JDK 21 虚拟线程支持
- 同步代码风格，异步性能
- 适合 I/O 密集型场景

### 3. 插件化架构
- 责任链模式处理请求
- 可插拔的插件机制
- 内置 WAF（Web 应用防火墙）、HTTP 转发插件示例

### 4. 灵活规则引擎
- 支持 AND/OR/NONE 匹配模式
- 可扩展的断言机制（13 种操作符）
- Header/URI/Cookie 等多种数据源

## 为什么选择虚拟线程

Lion Gateway 从响应式编程切换到虚拟线程，主要原因：

| 对比项 | Reactor | 虚拟线程 |
|--------|---------|----------|
| 代码复杂度 | 高 | 低 |
| 调试体验 | 困难 | 简单 |
| 学习成本 | 高 | 低 |
| 堆栈跟踪 | 复杂 | 清晰 |
| 兼容性 | 需要响应式库 | 可使用阻塞库 |
| 性能（I/O 密集） | 相当 | 相当 |

1. **代码更简洁** - 同步代码风格，易于理解和维护
2. **调试更方便** - 堆栈清晰，没有响应式流的复杂性
3. **兼容性更好** - 可以使用阻塞式库
4. **性能相当** - I/O 密集场景下性能与响应式相当
